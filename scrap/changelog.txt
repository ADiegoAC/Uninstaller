## [0.1.0] O Início
- **[Motor - 0.1.0]** Arquitetura base de retorno via `PSCustomObject`. Scan básico do Registro do Windows (Win32) e varredura nas 4 chaves padrão (`HKLM/HKCU` e `WOW6432Node`).
- **[Motor - 0.1.1]** Filtro inicial de componentes do sistema (`SystemComponent`) e chaves pai (`ParentKeyName`).
- **[UI - 0.1.0]** Esboço inicial da janela principal e DataGridView.
- **[Integração - 0.1.0]** Primeiro Dot-Sourcing (`. .\Un1nst4ll3r.ps1`) chamando a função de scan e preenchendo o Grid de forma bruta.

## [0.2.0] Microsoft Store (AppX)
- **[Motor - 0.2.0]** Integração com a Microsoft Store (AppX) via `Get-AppxPackage`. Filtro de Frameworks base (`IsFramework -eq $false`) e apps não assinados (`SignatureKind -ne "None"`).

## [0.3.0] Filtros de System Apps
- **[Motor - 0.2.1]** Filtro de Aplicativos Invisíveis do Sistema (Oculta apps da Microsoft onde `AppListEntry = none`). Filtro de Componentes Nativos e filtro de GUIDs puros (ex: `1527c705-...`) para remover ruído no Grid.

## [0.4.0] Fallback Fabricante e Design Base
- **[Motor - 0.2.2]** Fallback de Fabricante: leitura de metadados do arquivo `.exe` via `[System.Diagnostics.FileVersionInfo]` quando o registro não fornece o `Publisher`. Limpeza de strings de fabricantes de apps da Store (remoção do prefixo `CN=`).
- **[UI - 0.2.0]** Criação do design base: Painel de cabeçalho, barra de ações e rodapé (Status Bar). Truque de DPI Awareness (`SetProcessDPIAware`) via C# embutido para evitar embaçamento em monitores com zoom.

## [0.5.0] Deep Size e Banco MSI
- **[Motor - 0.3.0]** Função `Get-Un1nst4ll3rDeepSize` (Fase 2). Cálculo de tamanho físico em disco via `Get-ChildItem -Recurse -Force`. Variável de ambiente `Local` adicionada ao retorno do Motor para suportar cálculos e checagens futuras.
- **[Motor - 0.3.1]** Consulta ao Banco de Dados MSI via COM Object `WindowsInstaller.Installer` para ler `InstallLocation` e `InstallSource`. Filtros Anti-Lixo MSI (Package Cache, TEMP, IXP, pastas hexadecimais). Roubo de Palavra-chave do Cache e adição de variáveis de ambiente nativas e caminhos críticos (`$env:LOCALAPPDATA\Programs`, `$env:SystemDrive`).

## [0.6.0] Metadados, Exportação e Ordenação
- **[Motor - 0.3.2]** Metadados Expandidos: Propriedades `InstallDate`, `HelpLink`, `UninstallString`, `NoRemove`, `NoModify` e `NoRepair` extraídas do Registro e retornadas no objeto.
- **[Integração - 0.2.0]** Exportação de JSON: A UI gera automaticamente o arquivo `Un1nst4ll3r_ScanResult.json` após o scan, permitindo análise forense externa e debug de heurística.
- **[UI - 0.2.1]** Auto-Resize da coluna Local com `AutoSizeMode = Fill` e `MinimumWidth = 150`. Controle Manual de Ordenação via variáveis de estado para grids não vinculados a bancos de dados.

## [0.7.0] Heurística, Atalhos e Órfãos
- **[Motor - 0.4.0]** Heurística de Caminho (Fases 3 e 4) para apps sem `InstallLocation`. Parser avançado do `UninstallString` (Regex para parâmetros do InstallShield antigo) e Parser de Atalhos (`.lnk`) do Menu Iniciar e Desktop via `WScript.Shell`. Detecção de programas órfãos (pasta inexistente no disco).
- **[Motor - 0.4.1]** Bug de codificação (Encoding UTF-8 vs Windows-1252): alteração de "Órfão" para "Orfao" para evitar quebras de parser em sistemas Windows em português.
- **[UI - 0.2.2]** Coloração de linhas de programas órfãos em vermelho no Grid.

## [0.8.0] Arquitetura de Bytes e Ordenação
- **[Motor - 0.5.0]** Mudança de paradigma: A propriedade `Tamanho` agora retorna `[long]` (Bytes puros) ou `$null` ao invés de strings ("10 MB"). (Breaking Change para ordenação matemática).
- **[UI - 0.3.0]** Refatoração do preenchimento do Grid para suportar o recebimento de Bytes. Formatação dinâmica na inserção de dados: Conversão de Bytes para KB/MB/GB ou "N/A".
- **[UI - 0.3.1]** Exceção `System.ArgumentException: Objeto deve ser do tipo Int64` ao clicar no header de Tamanho. Resolvido com a criação de coluna oculta `TamanhoBytes`, separando dado de exibição do dado de ordenação.
- **[Integração - 0.3.0]** Evento `Add_ColumnHeaderMouseClick` implementado para interceptar cliques no header e ordenar pela coluna oculta de Bytes.

## [0.9.0] SplashScreen e Injeção de Dependência
- **[UI - 0.4.0]** Implementação da SplashScreen sem bordas para feedback visual durante o carregamento.
- **[Integração - 0.4.0]** Injeção de dependência de Log: a UI define o `$Global:Un1LogAction` antes do scan. Uso estratégico de `DoEvents()` para evitar congelamento da UI síncrona, permitindo atualização em tempo real da SplashScreen.

## [1.3.0] PS7 Auto-Updater e Ciclo de Vida do Runtime
- **[UI - 1.3.0]** Sistema de Verificação e Auto-Atualização para PowerShell 7. Prompt estilizado ("Dark Mode") solicitando ao usuário a atualização, com verificação de existência do Winget.
- **[UI - 1.3.1]** Mini-Terminal de Instalação Winget: Formulário estilo terminal cyborg para exibição do progresso de instalação em tempo real. Atualização do stdout sem congelar a thread principal da UI usando `DoEvents()`.
- **[UI - 1.3.2]** Parser de ANSI e Progress Bars do Winget: Filtragem de códigos de escape ANSI (`\x1b`) e tratamento inteligente de Carriage Returns (`\r`). Detecção via Regex de spinners e porcentagens para sobrescrever a linha atual em vez de criar rolagem poluída.
- **[UI - 1.3.3]** Auto-Relançamento Inteligente (Auto-Relaunch): Após instalação bem-sucedida, a UI localiza o novo `pwsh.exe` (via PATH ou fallback em pastas conhecidas) e reinicia o script automaticamente sob o novo runtime.
- **[Integração - 1.3.0]** Garantia de ciclo de vida: O App só inicializa o Motor e os caches pesados após validar que está rodando sob o runtime correto (PS7+), prevenindo bugs de compatibilidade de módulos (como o `Appx`).
